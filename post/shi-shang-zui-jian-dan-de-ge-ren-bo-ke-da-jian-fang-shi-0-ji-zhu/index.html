史上最简单的个人博客搭建方式(0技术)

2020-03-10

本次春节号称史上最烂的春节。

在老家农村宅了近50天，老家是武汉的，风暴的中心。

每天看到那些不断上涨的数字，自己一条条鲜活的生命离世，非常的压抑。

------ 扯远了。

这种状态下，人生都思考了好多次，总想写点什么。于是张罗着自己弄个博客，也不需要太花哨，不需要非常多的功能。

第一时间想到了GayHub的 page 功能，毕竟我又穷又没有技术，花钱的有点心疼，太复杂的搞不来，于是开始折腾

一开始是用 Jekyll + Github Pages 的方案，但是折腾来折腾去，总不是很满意，尤其更改了某个中意的主题之后，再push，页面就打不开了，非常之诡异；我把他归结为我技术不行，------ 我技术确实不行，没接触过前端。

后来又改用hexo+GitHub的方案，这次非常顺利，但是hexo -d 的命令有一个非常致命的问题，他只会 推送 _post，其他的改动不管(比如我动不动，可能会改点主题的布局、内容之类的)，也不知道是我研究的不对还是怎么的。

再次抛弃了hexo -d 的命令，打算直接用 Git push 的命令了。

某天刷头条，头条居然给我推送了一篇 搭建静态博客的文章(我始终怀疑头条是在疯狂的搜集个人隐私)，提到了一个神器 Gridea，我一看就喜欢上了他，对于我这种懒人没有比这个更好的东西了，难怪人说懒惰推进社会进步。

事前准备

  1. GitHub 账户，先注册一个GitHub的账户，如何注册就不说了：github.com
  2. 下载Gridea 神器并安装：gridea

万事大吉，准备开工。

开始搭建

一. GitHub设定

1. 获取token

登录 GitHub 之后，创建一个 Token，勾选上 repo 的相关权限即可。生成之后记得把 Token 复制到你的本地，因为一旦关闭网页将不能再看到它。

点击这里可以进入 创建token的页面：创建token

勾选 repo相关的几个权限即可，完成后提交，可以得到一个 token的 字符串。

将这个字符串，复制下来，存到本机的一个记事本。

2. 创建仓库
接下来就是创建仓库，存放构建后的网站文件。创建一个公开仓库，名为 用户名.github.io。（将用户名替换为你的 GitHub 用户名）

离成功已经很近了！

二. Gridea设定

1. 远程连接设定

打开 Gridea 进行远程设置。按照下图的示例进行设置，然后点击「检测远程链接」，进行配置检查。

这一步就是设定Gridea这个工具与刚注册的GitHub账户、页面之间的连接，请按图上说明进行设定。

设定完成后，点击最左侧的 【同步】按钮，将内容同步到GitHub，你的博客就创建成功了。可以点开

https:// xxxx.github.io 这个域名查看 ，其中的 xxxx 换成你的GitHub 用户名

2. 主题设定

Gridea提供了许多的主题，可以下载到本机，本机主题和源码存储的目录应该是

C:\Users\xxxx \Documents\Gridea\themes

其中的 xxxx 换成你的本机的用户名，或者从左下角，进入 Gridea 设置，

找到站点源文件路径

所有下载的主题，都可以放到这里，然后重启Grid后就能看见。

3. 发布文章

Gridea支持所见即所得的撰写方式，也支持MD语法。新增完文章后，点击同步，等同步完成后，打开你的博客网址，即可看到新增的文章。
