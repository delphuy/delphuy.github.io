0106 学习python 的第2个坑--第三方包与pycharm

本文最后更新于：24 分钟前

ide 我用的是pycharm，使用cmd 安装xlrd模块，提示安装成功

然而在pycharm 里 import 的时候，却是红线提示

有点懵逼，明明是已经安装成功了啊；
于是我使用pycharm的解释器再安装一遍
显示成功，我寻思了一下，应该是解释器 的问题，我在创建项目时没有指定python的解释器路径。
于是重新进入 setting–> project –> interpreter，选择我安装目录的python解释器：

果然不出我所料，这里是 空白的，显示 ：No interpreter，于是我选择 show all，然后选择看起来是正确的那个解释器，然后点 OK：

然后，新的问题出现了，显示 ** Python packaging tools not found. Install packaging tools **，坑爹啊！
然后，点击 install XXX，安装这个 packing tools，又报错了：

看提示，说是找不到这个SDK，头秃啊！！

我突然看到这个提示里的 版本， 3.7 ！以及路径，貌似我和安装的不一样，可能是之前的残留；
于是，我感觉我找到问题的根源了，再次进入 setting–> project –> interpreter –> show all 将这个3.7 的 解释器 点击右边的 - 号 移除，然后点击 + 添加 我安装的 3.9版本的解释器，一顿操作猛如虎：

点击OK，返回解释器的界面，perfect！！！一切正常，我之前安装的第三方包 都正确显示：

再会到 pycharm 的编辑界面， import 一下试试：

已经有智能提示了；

我突然想到，其实在创建项目的时候，就可以直接选择正确的解释器，这样用pip 命令安装的包，在pycharm就能直接使用了
